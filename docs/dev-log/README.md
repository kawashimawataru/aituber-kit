# 開発ログ（dev-log）

日付付きの作業記録と、**常に最新の進捗**（STATUS）を管理する。

---

## ファイル

| ファイル               | 役割                                                 | 更新頻度                   |
| ---------------------- | ---------------------------------------------------- | -------------------------- |
| [STATUS.md](STATUS.md) | **現在の Phase・タスク・ブロッカー・次のアクション** | 作業のたびに               |
| `YYYY-MM-DD.md`        | その日の実施内容・決定・メモ                         | 日単位（セッション終了時） |

---

## 日次ログの書き方（テンプレート）

新規ファイル `docs/dev-log/YYYY-MM-DD.md`:

```markdown
# 開発ログ YYYY-MM-DD

## サマリー

- （1〜3 行で今日やったこと）

## 実施タスク

| ID  | 内容                              | 結果 |
| --- | --------------------------------- | ---- |
| 0-1 | 例: Node 24 手順を dev-log に追記 | 完了 |

## 決定・メモ

- （設計判断、調査結果、ユーザー要望）

## ブロッカー

- （なければ「なし」）

## 翌日 / 次セッション

- [ ] （具体的な 1〜3 タスク）
```

---

## STATUS の書き方

[STATUS.md](STATUS.md) は **常に 1 画面で把握できる** よう、長文は日次ログへ逃がす。

必須フィールド:

- `アクティブ Phase`
- `進行中タスク`（タスク ID リスト）
- `完了済み（直近）`
- `次にやること`（優先順）
- `ブロッカー`

---

## 計画書との関係

- 計画の正本: [../project/](../project/)
- ログに「Phase 3 は GO と判断」などが出たら、必要に応じて `project/phases/phase-03-memory.md` か `current-state.md` も更新する
